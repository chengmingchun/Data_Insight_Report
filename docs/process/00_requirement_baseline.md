# 需求基线

## 基线信息

- 基线来源：`AI_Data_Insight_Report_SDD.md` v1.0
- 基线日期：2026-07-15
- 实现方式：模块化单体、标准 SDD 生命周期、TDD 前移
- 第一版输入：电商订单 CSV
- 第一版输出：页面分析、交互图表、清洗/异常 CSV、离线 HTML 报告

## 已确认口径

1. 销售额、平均订单金额、地区/品类/商品销售额与销量只统计状态为“已完成”的有效订单。
2. 退款率 = 已退款订单数 / 通过关键业务校验的订单数。
3. 非法日期属于部分有效数据：参与非时间类统计，不参与每日趋势。
4. 缺失地区填“未知地区”，缺失品类填“未分类”，缺失折扣填 0。
5. 重复订单保留第一条；其余记录进入拒绝集并保留原因。
6. 关键字段缺失、数值无法解析、非正数量/价格、越界折扣进入拒绝集。
7. IQR 只标记值得关注的统计异常，不从销售统计中删除。
8. LLM 只接收聚合 JSON，不读取原始 CSV，不计算任何指标。
9. 默认 LLM Provider 为 DeepSeek；设置从前端设置页输入，密钥仅驻留当前会话。
10. DeepSeek 未配置或调用失败时，自动使用模板摘要，统计、图表与导出不受影响。

## 范围控制

本次实现 SDD 必做项及低风险加分项：二维透视、IQR、异常下载、数字白名单校验、交互图表、完整测试。PDF、数据库、日志文件、通用字段映射不进入第一版。
